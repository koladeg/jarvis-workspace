# workspace-research

This folder is Robin's lean working area.

## What should stay in git
- durable research notes in `memory/*.md`
- small text-based trackers or summaries
- stable config needed to reproduce the workflow

## What should not pile up here
- generated outputs in `output/`
- temporary consolidation inputs like `memory/consolidation-input-*.txt`
- logs and scratch files

## Housekeeping rule
Keep the research workspace note-first and retrieval-friendly:
- preserve useful daily notes
- archive/summarize when patterns become repetitive
- avoid committing bulky generated artifacts unless they are explicitly worth keeping
